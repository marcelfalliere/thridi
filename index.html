thridi


WebGL / Three.js editor with hot code reload.

Please, have a look.


TL;DW : demo of a three.js editor. Updating the code on the left part of the screen refreshes the scene on the right part without loosing state (what we call "hot-reload"). Find out more on the explained blog post

Now let's talk features.

Standards compliant

We love the web and Three.js. The code produced in Thridi is plain-old javascript and will work on the modern web (where webgl works, you know ?). It is not tied to a hot-reload technology like closure script, react components, or proprietary framework. In fact any example from the three.js website can be imported in thridi and you can start playing coding. The code here is standard web!

Hot reload

That means materials, geometries, camera, controls, animations (everything in the scene) : they all get to the right state as you keystroke. Fine tuning the lights, the texture, the colors, the controls will never be this easier.This is the key feature of thridi. Check out the video above for a demonstration.

Shaders

A shader is a small piece of code written in GLSL to run on the GPU. It has become the crux of the matter in 3D. Ok. But ! working with shaders in Three.js is just painfull. With Thridi, forget about the ugly <script id="fragment-shader">... or the bad THREE.ShaderLib[ 'awesomeShader'] ... Edit the shader in a separate file. You will still get hot reload. This is the good way.

IDE for your everyday three.js's needs

Thridi aims at answering your everyday needs when it comes to three.js development. Are your making a 3D background that moves as the user scrolls in the page ? Do you use the webcam ? The mic input ? Or simply you don't want to use Unity and the like, and have an integrated work environment made by a three.js dev for three.js dev ?
