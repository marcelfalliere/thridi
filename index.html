thridi

WebGL / Three.js editor with hot code reload.

Have a look.

As you can see, it is regular code on the left and a 3D scene on the right. The 3D scene reloads without loosing states as you type. Voila.

Standards compliant

We love the web and three.js. The code produced in thridi is plain-old javascript and will work on the good-old web. It is not tied to a hot-reload technology like closure script, react components, or stuff like that. In fact any example from the three.js website can be imported in thridi and you can start playing coding. The code here is standard web! Just one thing : everything that gets added to a scene needs an id.

Refresh without loosing state

That means materials, geometries, camera, controls and animations all get to the right state as you save. Fine tuning the lights, the texture, the colors, the controls will never be this easier.

Also, shaders

Working with shaders in three.js is just painfull. While a shader is a piece of code, it is send as plain text. So forget about <script id="fragment-shader">... or THREE[Libs]... the shader gets hot reload too, and you store them in a file that makes work and maintenance easier.
